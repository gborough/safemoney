val unsafe_decimal_to_q : decimal:string -> sep:Separator.t -> (string, string) Stdlib.result

unsafe_decimal_to_q decimal sep parses the string representation of a float value with printing configuration e.g. let sep = Separator.make_exn ('.', Some ',') in unsafe_decimal_to_q ~decimal : "+123,456.789" would convert value to "123456.789" It is unsafe in a sense that the origin of float is deemed to be unverified by default and might be as a result of lossy operations
